Planetz Agent Deck はローカル向けの操作デッキ。 課金プラン Planetz Cloud では、クラウド上の仮想環境にワークスペースを載せ、 ブラウザだけでタスク投入・監視・ワークフロー修正まで行えます。

Open Source

Planetz Agent Deck

手元の Mac / Linux / Windows で、リポジトリ内のエージェント実装をひとつのデッキに集約します。

タスクレーン & エージェント盤

待機・実行中・完了をひと目で把握。エージェントごとの稼働と、タスクの流れを同じ画面で追えます。

ワークフロー GUI

ワークフローをフォームやフロー図で編集。保存前に検証し、壊れた設定をその場で直せます。

Retry / Resume / Watch

失敗した実装の再実行、ログの追跡、常時監視のオン／オフをデッキから操作できます。

同梱 Orbit のみ

別途 CLI を入れなくても動きます。同梱の Orbit で、チーム全員が同じ実行環境を共有できます。

リポジトリは MIT 想定の OSS。ビルド・実行は guilz-dev/planetz-release を参照してください。

Planetz Cloud · サブスクリプション

クラウドの仮想環境を、
ブラウザのデッキで操作

ローカルに環境を構築できないメンバーでも、チーム共通の VM 上で同じワークスペースを共有。 実装の投入・監視・設定変更を Web から行い、成果物は Git リポジトリに残します。

  • マネージド VM — リポジトリの clone、Orbit の常時監視、デッキ用データをクラウド側で維持
  • ブラウザコントロール — Deck と同型の UI を Web で提供（タスク・ワークフロー・接続状態）
  • 修正はクラウド上で — ワークフローやプロジェクト設定の編集・検証を VM 内で完結
  • シークレットは VM 内 — デッキ方針どおり、API キーを SaaS に預けない設計を維持

How it works

Deck 無料、
Cloud で「場所」を買う

  1. 01

    Deck（OSS）でローカル検証

    手元のワークスペースを開き、タスク投入と watch の挙動を確認。チームの標準フローを固めます。

  2. 02

    Cloud に workspace をプロビジョン

    ブランチ / リポジトリを指定して VM を起動。Orbit がタスクを監視し、実装の実行ログを蓄積します。

  3. 03

    ブラウザから実装をコントロール

    プロンプト投入、ワークフロー修正、失敗タスクの retry — 操作は Cloud UI、実行は VM 内の Orbit。

  4. 04

    成果を Git に戻す

    ワークフローやエージェント設定の変更は VM 上で検証後、いつもどおり PR で本番リポジトリへ。

FAQ

よくある質問

他のプロダクトとどう違うのですか？
多くのマルチエージェント製品は、IDE やクラウド上の セッション管理と実行状況の可視化が中心です。Planetz Agent Deck はそれに加えて、 投入・監視・失敗時の再実行・設定変更までの運用一式を、同じ画面で完結できるように設計しています。 実行は内部のハーネス／オーケストレーターが担当し、変更はワークスペース内で Git に残ります。 API キーは SaaS ではなく VM / ローカルに保持する前提のため、運用とデータ管理の主導権をチーム側に置けます。
Planetz とは何ですか？
エージェント実装の「司令室」です。タスクの投入・進捗・失敗対応をデッキ UI に集約し、 裏側では同梱の Orbit がワークスペースを実行します。Planetz 単体の製品として提供しています。
Deck だけ使う場合、課金は必要ですか？
不要です。Deck は OSS としてローカル利用を想定しています。Cloud は任意のサブスクリプションです。
Cloud で何が「修正」できますか？
タスクの投入、ワークフロー・プロジェクト設定の変更、失敗時のやり直しなど、Deck と同等の操作をブラウザから行う想定です。 編集と検証は VM 内で完結し、変更はリポジトリに残ります。
ローカル Deck と Cloud の関係は？
同じ操作画面と手順を共有します。ローカルで慣れたチームが、そのまま Cloud 上の workspace に移行できることを目指しています。
