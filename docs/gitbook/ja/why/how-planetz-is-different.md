# Planetz の独自性

> Status: なぜ Planetz か · 対象バージョン 0.1.x

## このページでわかること

すでに知っているツール——チャット型アシスタントや「誰でも作れる」アプリビルダー——の隣で Planetz がどこに位置するか、なぜ本当の強みが**コード生成ではなく統治**にあるのか、そしてエッジ AI と **Manta** デスクトップロボがどう噛み合うかを説明します。

---

## 市場は「AI がコードを書ける」を解いた。「整合させ続ける」は誰も解いていない

Cursor、v0、Bolt、Lovable、Replit Agent といったツールは、AI がソフトウェアを作れること、そして非エンジニアでも実際に動くものを出荷できることを、すでに証明しました。その競争はほぼ決着がついており——彼らが得意とすること、すなわち*コードの生成*は、コモディティになりつつあります。

しかし、どのツールも規模が大きくなると同じ壁に当たります。システムが成長するにつれ、チームが実際に決めたことから逸れていき、それを照合し続けるツールが存在しないのです。**この空き地——成長する AI 製システムを蓄積された意図と整合させ続けること——こそ Planetz が戦う場所です。**

## チャット型アシスタントとの違い

パーソナル AI アシスタントは 1 つのチャット窓に住み、*あなたを記憶しよう*とします。その「記憶」は創発的かつ私的で——便利ではありますが、監査も差分比較もチーム共有もできません。

Planetz は別の形をしています。良い記憶を持つ 1 体のアシスタントではなく、**統治されたプロセス**を走らせます——検査・バージョン管理・共有できるワークフローに従う、エージェントの艦隊です。価値は「記憶するアシスタント」ではなく、**信頼でき再現できるプロセス**にあります。

| | チャット型アシスタント | Planetz |
|---|---|---|
| 作業の単位 | 1 つの会話 | 並列に走る多数のタスク |
| 「記憶」 | 創発的・私的 | 明示的な意図の記録、監査可能 |
| 制御 | モデルを信じる | 統治されたワークフロー＋人間ゲート |
| チーム適性 | 個人向け | 共有・差分・再現が可能 |

## 「誰でも作れる」ツールとの違い

これらのツールは作業を前倒しします——*良いプロンプト／仕様を書けば、こちらが作ります*。しかし、完全な仕様を先に書くこと自体が、難しい専門スキルです。だから実際には結局エンジニアが必要に戻りますし、エンジニアですら、システムが存在する前に完全に仕様化することはできません（仕様は作りながら*発見される*ものです）。

Planetz はこれを反転させます。完璧な仕様を予測するよう求めません。**挙動が現実になった瞬間に**意図を捕まえます——エージェントがやったことを見て、それが望んだものかを*見て分かる（recognition）*。この「見て分かる」スキルはほぼ誰もが持っています。それが要点です。**すべてのオペレーターがシニアエンジニアであることを前提にせずに**、結果を保守可能で仕様どおりに保てるのです。

## 本当の堀：生成ではなく統治

Planetz の背後にある最も深い発見はこうです。エージェントにプロンプトを与えるとき、モデルは衝突に気づけないほどバカなわけではなく、たいていは**あなたに表面化させず、自分で黙って決めて局所パッチを当てる**のです。ハーネスの指示が 1 行あるだけで、弱いモデルも強いモデルも例外なく、「黙って決める」から「止まって人間に裁定を渡す」へと切り替わります。

つまり持続的な強みは、AI を賢くすることではありません。モデルがただではくれない 2 つのものです。

1. **決定の統治（decision governance）** — 重要な瞬間を、黙った推測ではなく明示的な決定点に変える。
2. **意図の記録** — それらの決定を蓄積しロックし、*次の*実行が再発見しなくて済むようにする。この記録はあなたのプロジェクト固有のもので、モデルの重みには決して焼き込めません。だからこそ、他のすべてがコモディティ化しても価値が残ります。

## しかも、あなたの望むやり方で走る

仕上げに、3 つの実用的な差別化点があります。

- **🔌 エッジ AI — あなたのコードはあなたのもの。** Planetz は、自分のマシン上のローカルモデル（Ollama 経由）で推論を走らせられます。だからソースコードや社内文書を公開 API へ出す必要がありません——プライバシー・セキュリティに有効で、オフライン作業も可能です。*（一部のツールを使うワークフローステップは依然クラウドプロバイダを利用します。ワークフローごとに選べます。）* より馬力が要るときは、**Planetz Cloud** が同じ Deck とワークフローのままスケールします——学び直し不要。
- **🐠 Manta — デスクトップロボ（任意）。** Manta は手のひらサイズの小さなコンパニオン機器で、承認疲れに物理的に答えます。状態 **LCD** と **アンビエントライト**が各エージェントの状態を一目で示し（`PROCESSING…`、`APPROVE?`、`ERROR`）、モニターを占有しません。専用の **Approve / Deny** ボタンで、リスクのある操作に意図的な 1 回の「GO」を出せます。**ホールド・トゥ・トーク**ボタンは、話した言葉をエージェントの次の指示に変えます。Planetz は Manta なしでも完全に動きます——Manta は制御をより穏やかで周辺的（ambient）にするだけです。
- **🌳 Git ネイティブ。** すべての実行は隔離され、差分が取れ、復元できます。コードは意図から再生成できる派生物として扱われるからです。

> _オープンソースの上に。_ Planetz のハーネス（アプリ内表示名 **orbit**）は、オープンソースエンジン [takt](https://github.com/nrslib/takt) の中核アイデアを採用し、その上に改良を重ねています——Deck の UI、意図の記録、ワークフロー自動ルーティング、エッジモデル、Manta を加えています。Planetz は独立したプロジェクトであり、takt の作者による関連・保守を受けるものではありません。
