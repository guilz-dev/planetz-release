# 今後の展望

> Status: ロードマップ（将来計画） · 対象バージョン 0.1.x

## このページの内容

Planetz の進む先。このページは**意図的に将来を見据えた**ものです——今日出荷済みのものもあれば、開発中・計画中のものもあります。完成品として語るのではなく、どれがどれかを明記します。

---

## 一貫する筋

Planetz の賭けは、エージェントがより自律的になるほど、希少になるのはコードではなく**エージェントが作っているものを掌握し、理解し続けること**だ、というものです。このロードマップのすべてがそれに資します——より多くの自律性を、より多くの理解と統治と対にする。

## Manta — デスクトップロボ

> **状態：リファレンスハードウェア、予約／ウェイトリスト。**

制御をモニターから切り離す、手のひらサイズのコンパニオン機器。各エージェントの状態を示す状態 **LCD** と **アンビエントライト**、意図的な確認のための物理 **Approve / Deny** ボタン、話した言葉をエージェントの次の指示に変える **ホールド・トゥ・トーク**。**USB-C / BLE** で接続し、オープンなデバイスプロトコルのロードマップを持ちます。Planetz は Manta なしでも完全に動きます。Manta は制御をより穏やかで物理的にします。

## Planetz Cloud

> **状態：計画中／ウェイトリスト。**

ブラウザから操作するマネージドのプライベートノードで、デスクトップアプリと**同じ Deck の UX とワークフロー**を持ちます。エッジモデルでローカルに始め、より馬力が要れば——製品を学び直すことなく——ワークスペースを Cloud へ移し、**チーム共有と監査ログ**を加えます。ローカル Deck と Cloud は、2 つの製品ではなく 1 つのスペクトラムの両端として設計されています。

## より多くの自律性を、安全な段階で

ロードマップは実行を**自律性のはしご**——*レポートのみ*から、*変更を提案し人間がマージ*、*安全な範囲内で無人実行*へ——上らせます。各段の昇格は、希望でオンにするのではなく、測定された信頼性で勝ち取ります。停止条件、承認ゲート、そして[意図台帳](../concepts/intent-ledger.md)が、その下の制御として残り続けます。

## 自分自身を改善するプロセス

> **状態：方向性であり、未出荷。**

今日、ワークフローと **facet**（再利用可能なペルソナ／ポリシー／知識の部品）はあなたが書きます。方向性は、これら*プロセス*が持続的なチーム資産へと育ち——そして司書のような助手がそれらを腐らせない（未使用を退役させ、重複を統合する）ことです。正直な言い方をすれば、価値は「AI が魔法のように自己改善する」ことではなく、**「あなたが築いた資産が劣化しない」**ことです。

## 理解度ダッシュボード

> **状態：計画中——私たちが最も重視する差別化点。**

自律エージェントの最大のリスクは**理解負債（comprehension debt）**です——誰も理解できないほど速くコードが出荷され、ついには誰も安全に変更できなくなる。計画中のダッシュボードは、その負債を*測定可能*にします——revert 率、生成と review のペース、週次ダイジェストといった指標——ので、エージェントが忙しいかどうかではなく、チームがまだ自分のシステムを理解しているかを見られます。

## これは何で*ない*か

Planetz があらゆるミスを防ぐとも、賢いモデルが統治を不要にするとも主張しません。むしろ逆です——エージェントが賢くなるほど黙って多くを決めるので、意図・決定・理解を掌握下に置く価値は増すばかりです。
